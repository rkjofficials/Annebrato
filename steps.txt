- **67.0000**: Service call required error - Contact HP Service. Printer may need repair
- **68.0000**: Imaging unit error - Check imaging unit position. May need replacement
- **69.0000**: Excessive temperature - Allow printer to cool. Check for blocked ventilation

## General Printer Troubleshooting

### Print Quality Issues

- **Faded printing**: Low toner/ink. Replace cartridge. Check fuser temperature
- **Streaks on page**: Dirty pickup roller or print drum. Run cleaning cycle. Clean rollers
- **Color misalignment**: Run color calibration. Check registration settings
- **Ghosting (faint image)**: Worn imaging drum or developer. May need replacement
- **Rough texture**: Check paper type. Verify toner fuser setting. Clean rollers
- **Toner smudging**: Fuser temperature too low. Check thermal settings. Inspect fuser

### Paper Feed Problems

- **Multiple sheets picked**: Pickup roller worn or dirty. Clean and replace if needed
- **No paper picked**: Check tray sensor. Verify tray positioning. Clean rollers
- **Skew (diagonal feed)**: Check paper guides. Verify tray alignment. Clean feed path
- **Duplex failure**: Check duplex unit connections. Verify duplex sensor

### Communication Errors

- **USB not recognized**: Try different USB port. Update printer drivers. Restart both devices
- **Network unreachable**: Check ethernet/WiFi connection. Verify printer IP address
- **Slow printing**: Check network load. Reduce print job complexity. Update firmware
- **Print job stuck in queue**: Restart print spooler. Clear printer queue. Power cycle printer

### Maintenance Required Messages

- **Toner low**: Order replacement toner immediately. Monitor for quality degradation
- **Waste box full**: Empty or replace waste container before continuing
- **Maintenance interval reached**: Perform maintenance on fuser, rollers, etc per manual
- **Drum replacement due**: Order replacement imaging drum. Schedule maintenance

# Opera OXI (OPERA Xchange Interface) - Comprehensive Error Documentation

## OXI Overview

OXI is the OPERA Xchange Interface that enables integration between OPERA PMS
and external systems (such as booking channels). This section contains ALL error
messages extracted from official Oracle documentation (Appendix A, C, E).

## OXI Message Direction

Messages are issued in two directions:
- **OPERA > External (Upload)**: Message sent from OPERA to external system
- **External > OPERA (Download)**: Message received by OPERA from external system

## OXI System Error Log - Detailed Error Reference

### Database and Connection Errors

- **Error**: Can't update thread status. Error: ORA-01012: not logged on
  - **Direction**: OPERA > External
  - **Reason**: Network connection is down. Database was restarted without stopping OXI service
  - **Solution**: When server is back up, stop/restart OXI service to refresh communication

- **Error**: Value of application parameter UP_PROC_IDLE_TIME in group OXI_GENERIC not found
  - **Direction**: OPERA > External
  - **Reason**: Upload processor timing parameter is missing, using default value (2)
  - **Solution**: Verify parameter settings in OXI configuration

- **Error**: B44 OXI Heart Beat failed. Error: ORA-01012: not logged on
  - **Direction**: OPERA > External or External > OPERA
  - **Reason**: Interface is not communicating with IFC Monitor (ORS interfaces with remote ORS)
  - **Solution**: Restart OXI service and verify database connectivity

- **Error**: ORA-04068 / ORA-04061 / ORA-04065 / ORA-06508: Package state discarded/invalidated
  - **Direction**: OPERA > External
  - **Reason**: Invalid objects in OXI schema after installation/upgrade. Processor not restarted
  - **Solution**: Check for invalid objects. Restart processor service after upgrade

- **Error**: ORA-04031: Unable to allocate shared memory
  - **Direction**: External > OPERA
  - **Reason**: Invalid objects in OXI schema. Shared pool exhausted
  - **Solution**: Check for invalid objects in schema. Restart processor

- **Error**: Can't connect to database. Error: ORA-01017: invalid username/password
  - **Direction**: OPERA > External
  - **Reason**: Database connection credentials are incorrect for OXI service
  - **Solution**: Verify OXI database user/password in registry settings (Windows) or config

- **Error**: ORA-01012: not logged on (during batch outbound or message dequeue)
  - **Direction**: OPERA > External
  - **Reason**: Connection to database not available. OXI service not running
  - **Solution**: Check OXI service status. Stop/restart service to refresh processors

- **Error**: ORA-00028: your session has been killed
  - **Direction**: OPERA > External
  - **Reason**: OXI service was 'killed' during database/app server restart
  - **Solution**: Restart OXI service once server is back up

- **Error**: ORA-01012: not logged on (Connection Error)
  - **Direction**: External > OPERA
  - **Reason**: Network disconnect or database restart without stopping OXI
  - **Solution**: Restart OXI service to refresh database connection

- **Error**: ORA-06502: PL/SQL numeric or value error
  - **Direction**: OPERA > External (Reservation)
  - **Reason**: Data conversion error during reservation processing
  - **Solution**: Verify OXI configuration and data format

# Opera PMS (Property Management System)

## System Error Log and Error Messages

Opera system errors are logged and can be viewed in the System Error Log. Common
errors and their causes are documented below. Error messages typically include a
description, cause, and suggested solution.

### Database and Connection Errors

- **ERR_DB_CONNECTION_FAILED**: Database connection failed. Verify Oracle database is running,
  network connectivity exists, and credentials are correct. Check database server logs.
- **ERR_DB_TIMEOUT**: Database query timed out. Query may be too complex or resources exhausted.
  Check database performance, run ANALYZE TABLE, rebuild indexes if needed.
- **ERR_DB_LOCK**: Database lock detected. Another user is accessing the same record. Wait
  for the lock to clear or restart the database connection.
- **ERR_DB_CORRUPTION**: Database corruption detected. Backup and restore from recent backup.
  Contact Oracle support if problem persists.

### Authentication and Authorization Errors

- **ERR_INVALID_CREDENTIALS**: Username or password invalid. Verify credentials are correct.
  Check that user account is active and not expired in Property Master.
- **ERR_PERMISSION_DENIED**: User does not have permission for this action. Check user role
  and menu permissions in Property Master > Users > Role Configuration.
- **ERR_ACCOUNT_LOCKED**: User account locked after multiple failed login attempts. Contact
  admin to unlock account or wait for automatic unlock (typically 30 minutes).
- **ERR_SESSION_EXPIRED**: Session expired due to inactivity. Log in again.

### File and Export Errors

- **ERR_FILE_NOT_FOUND**: File does not exist or cannot be accessed. Verify file path and
  that file exists. Check file/folder permissions.
- **ERR_FILE_WRITE_FAILED**: Unable to write to file (disk full, permission denied, or bad
  path). Check disk space, permissions, and file path syntax.
- **ERR_EXPORT_FAILED**: Export operation failed. Could be due to missing data, invalid
  format, or file system error. Check export log for details.
- **ERR_IMPORT_FAILED**: Import operation failed. Verify file format matches requirements.
  Check that file is not corrupted.

### Reservation and Booking Errors

- **ERR_RESERVATION_NOT_FOUND**: Reservation record not found. Verify confirmation number
  or guest name is correct.
- **ERR_ROOM_NOT_AVAILABLE**: Room is not available for requested dates. Check room status
  and availability in Rooms module.
- **ERR_RATE_NOT_AVAILABLE**: Rate not available for selected room type or dates. Verify
  rate setup in Rate Master.
- **ERR_DUPLICATE_RESERVATION**: Duplicate reservation already exists. Check for existing
  reservations before creating new one.
- **ERR_INVALID_DATES**: Check-out date must be after check-in date. Verify date ranges.

### Payment and Billing Errors

- **ERR_PAYMENT_DECLINED**: Payment was declined by payment processor. Verify card details,
  available funds, and payment gateway connectivity.
- **ERR_INVALID_AMOUNT**: Payment amount is invalid or exceeds maximum allowed. Verify amount.
- **ERR_FOLIO_NOT_POSTED**: Folio cannot be posted. Guest may have outstanding balance.
  Post outstanding charges or adjust folio details.
- **ERR_BILLING_FAILED**: Billing operation failed. Check billing settings and guest account status.

### Printer and Report Errors

- **ERR_PRINTER_NOT_FOUND**: Printer not found or not configured. Add printer in Settings >
  Printers. Verify printer is online and network accessible.
- **ERR_PRINT_FAILED**: Print job failed. Check printer status, paper, and ink/toner. Restart
  print spooler if stuck jobs exist.
- **ERR_REPORT_GENERATION_FAILED**: Report could not be generated. Verify report template
  exists and data is available. Check report parameters.

### Network and Connectivity Errors

- **ERR_NETWORK_TIMEOUT**: Network timeout/connection lost. Check network connectivity, firewall
  rules, and server availability. Restart application.
- **ERR_INVALID_URL**: URL/hostname is invalid. Verify server address is correct and resolvable.
- **ERR_SSL_CERTIFICATE_ERROR**: SSL certificate error (expired, self-signed, or invalid).

### Application and Configuration Errors

# Internet Connectivity

## Basic Connectivity Checks

- Check physical connection: verify cable is plugged in (green/orange light on
  port).
- Check interface status: `ip link show` (Linux) or `ipconfig` (Windows).
- Ping default gateway: `ping <gateway-ip>` (typically 192.168.1.1 or similar).
- Ping external DNS: `ping 8.8.8.8` (Google) or `ping 1.1.1.1` (Cloudflare).
- Ping by hostname: `ping google.com` to verify DNS resolution.

## Network Configuration

- Check IP address assignment: `ip addr show` (Linux) or `ipconfig /all`
  (Windows).
- Verify DHCP is enabled if IP is dynamic; check lease time and renewal.
- For static IP, verify subnet mask and gateway are correct.
- Check DNS servers: should show at least one (2 preferred).
- Test DNS: `nslookup google.com` or `nslookup 8.8.8.8`.

## Troubleshooting No Internet

- Restart network interface: `sudo systemctl restart NetworkManager` (Linux) or
  `ipconfig /release` then `ipconfig /renew` (Windows).
- Check router: ping router IP; restart router if no response.
- Check modem: restart modem (power cycle 2 minutes).
- Check WAN connection status on router admin panel (usually 192.168.1.1).
- Review router logs for WAN link errors.

## Wi-Fi Issues (if wireless)

- Verify network is visible: `nmcli device wifi list` (Linux) or list networks
  (Windows).
- Verify you are connected to correct SSID (not a guest network by mistake).
- Check signal strength: should be > -70dBm for good speed.
- Move closer to AP if signal is weak.
- Restart Wi-Fi radio: uncheck "Enable Wi-Fi" in settings, wait 10s, re-enable.
- Forget network and reconnect (forces re-authentication).
- Update Wi-Fi driver if connection is unstable.

## Firewall and Security

- Disable firewall temporarily to verify it is not blocking internet; leave
  disabled only for testing.
- Re-enable firewall after testing: `sudo ufw enable` (Linux) or Windows
  Defender Firewall.
- Add exception to firewall for applications that need internet access.
- Check for proxy configuration: if using corporate proxy, verify proxy settings
  are correct.
- Clear proxy settings if office network reconnected elsewhere.

## DNS Issues

- If websites load by IP but not by name, DNS is likely problem.
- Try alternative DNS: use 8.8.8.8 (Google) or 1.1.1.1 (Cloudflare).
- Flush DNS cache: `sudo systemctl restart systemd-resolved` (Linux) or `ipconfig
  /flushdns` (Windows).
- Check `/etc/resolv.conf` (Linux) or DNS settings in Network Adapter
  (Windows).
- Test DNS propagation: use online tool if recent DNS change made.

## Bandwidth and Latency

- Check WAN speed: use speedtest.net or iperf if available.
- Monitor latency: `ping -c 10 google.com` shows min/avg/max ms.
- Latency > 100ms indicates potential network issues.
- Check for heavy downloads or backups consuming bandwidth.
- Use netstat to check active connections: `netstat -an | grep ESTABLISHED`.

## VPN and Remote Connection

- If using VPN, verify VPN is connected and showing active tunnel.
- Check VPN logs if disconnecting frequently.
- Restart VPN client if intermittent connection loss.
- Verify VPN credentials if authentication errors appear.
- Test split tunneling configuration (some traffic local vs VPN).

# GDS (Global Distribution System) Error Handling

## Overview

GDS (Global Distribution System) property uploads are managed through the GDS
Property Upload feature. Errors during GDS uploads are tracked and can be
reprocessed after correction. This section contains all known GDS errors from
official Oracle documentation.

## GDS Upload Errors - Comprehensive Reference

### Property Configuration Errors

- **Property Upload Message Error**: Upload message failed to process
  - **Cause**: Property data is incorrect or incomplete
  - **Solution**: Check property data and correct in Status Update screen

- **GDS Property Configuration Missing**: Property not configured for GDS distribution
  - **Cause**: Property master setup incomplete
  - **Solution**: Set up property in GDS Property Upload configuration

- **Invalid Property ID**: Property ID does not exist or is invalid
  - **Cause**: Property identifier is wrong or doesn't match GDS system
  - **Solution**: Verify property identifier matches GDS records

- **Missing Required Property Data**: Required field missing in property upload
  - **Cause**: Property Master has incomplete data
  - **Solution**: Complete all required fields in Property Master

- **Property Data Validation Failure**: Property data failed validation rules
  - **Cause**: Data doesn't meet GDS specification requirements
  - **Solution**: Review data format and requirements. Correct values.

### Connectivity and Communication Errors

- **Pegasus Connection Error**: Connection to Pegasus failed
  - **Cause**: Network issue or Pegasus server offline
  - **Solution**: Verify network connectivity and Pegasus server status

- **GDS Server Unreachable**: Cannot reach GDS server
  - **Cause**: Network timeout or GDS server down
  - **Solution**: Verify GDS endpoint URL. Check firewall rules. Retry later.

- **Communication Method Not Configured**: GDS communication parameters not set
  - **Cause**: Missing GDS connection configuration
  - **Solution**: Configure GDS in Setup > GDS Property Upload > Configuration

### Rate and Inventory Errors

- **Rate Upload Failed**: Rate data failed to upload to GDS
  - **Cause**: Rate format invalid or currency mismatch
  - **Solution**: Verify rate setup and currency settings

- **Inventory Out of Sync**: Room inventory doesn't match GDS records
  - **Cause**: Rooms sold outside GDS without sync
  - **Solution**: Manually synchronize inventory. Check OXI for sync issues.

- **Rate Currency Mismatch**: Rate currency doesn't match GDS currency
  - **Cause**: Property currency different from GDS requirement
  - **Solution**: Verify and correct currency in rate setup

### Data Format Errors

- **Invalid Date Format**: Date in upload message doesn't match GDS format
  - **Cause**: Date format specification not followed
  - **Solution**: Use YYYY-MM-DD format or GDS-specified format

- **Invalid Field Value**: Field contains invalid data type
  - **Cause**: Text in numeric field or wrong format
  - **Solution**: Correct field value to match specification

- **Missing Mandatory Field**: Required GDS field is empty
  - **Cause**: Incomplete property setup
  - **Solution**: Complete all mandatory fields before upload

## GDS Error Handling Workflow

1. **Identify Error**: Setup > System Configuration > Channels > GDS Property Upload > Error Handling
2. **Review Details**: Click error to view full message and description
3. **Identify Root Cause**: Read error explanation and reason section
4. **Correct Data**: Go to Status Update screen, correct property data or configuration
5. **Reprocess**: Return to Error Handling, locate by Message ID, click Reprocess
6. **Monitor Status**: Check message status changes (ERROR  PENDING  PROCESSED)
7. **Verify Success**: Confirm property data is updated in GDS

## GDS Property Upload Status States

- **ERROR**: Upload failed and must be corrected before retry
- **PENDING**: Message is queued and awaiting reprocessing
- **PROCESSING**: Message is currently being uploaded to GDS
- **PROCESSED**: Message was successfully uploaded to Pegasus and distributed to GDS
- **RETRY**: Automatic retry is in progress for failed message
- **BLOCKED**: Manual intervention required; cannot be auto-reprocessed

## Property Master Requirements for GDS Upload

Verify these fields are complete for successful GDS upload:
- Property ID (must match GDS system)
- Property Name
- Property Currency
- Address and Country
- Property Type
- Check-in/Check-out times
- Mandatory Rate Setup
- Room Types and Inventory
- GDS Configuration/Codes

## GDS Troubleshooting Checklist

1. Verify GDS connection is active and authenticated
2. Confirm property ID matches between OPERA and GDS
3. Check rate currency matches property currency
4. Ensure all mandatory Property Master fields are populated
5. Verify room inventory is accurate and not oversold
6. Test with small data set before full upload
7. Check GDS system doesn't have duplicate properties
8. Review GDS upload logs if available from provider
9. Contact GDS provider (Pegasus/Sabre/Amadeus) for GDS-side errors
10. Escalate to Oracle Support if connection errors persist

# End-of-Day (EOD) Troubleshooting

## Overview

End-of-Day (EOD) is a critical daily process that finalizes all transactions,
posts folios, generates reports, and backs up the database. This section contains
comprehensive EOD errors and solutions.

## Common EOD Errors - Detailed Solutions

### Process Execution Errors

- **EOD Backup Failed**: Daily backup failed during EOD process
  - **Cause**: Backup destination doesn't exist or is inaccessible. Insufficient disk space
  - **Solution**: Verify backup path exists and has write permissions. Check free disk space (at least 50GB recommended). Correct path if needed.

- **EOD Posting Timeout**: EOD posting took too long and timed out
  - **Cause**: Too many transactions, slow database, or resource contention
  - **Solution**: Increase timeout in EOD configuration. Run EOD during low-traffic hours (2-5 AM). Optimize slow database queries.

- **EOD Process Interrupted**: EOD process was interrupted or cancelled unexpectedly
  - **Cause**: User closed application, network disconnect, or system crash
  - **Solution**: Do NOT interrupt EOD once started. Restart from beginning. Verify all transactions are rolled back.

- **Folio Posting Error**: Error posting guest folio during EOD
  - **Cause**: Outstanding balance, invalid GL account, or data integrity issue
  - **Solution**: Check folio balances. Post outstanding charges. Verify GL account configuration. Clear blocked transactions.

- **Report Generation Failed**: Daily reports could not be generated
  - **Cause**: Report template missing, file path invalid, or printer not available
  - **Solution**: Verify file path exists and is accessible. Check report templates exist. Verify default printer/export location.

- **Journal Entry Error**: GL journal entries failed to post
  - **Cause**: Invalid GL account, posting period closed, or accounting parameter issue
  - **Solution**: Check GL account setup. Verify posting period is open. Review accounting configuration.

- **Database Lock**: Database locked during EOD preventing process completion
  - **Cause**: Another process accessing database or previous EOD not completed
  - **Solution**: Kill other database connections. Rollback incomplete EOD. Restart EOD process.

- **Insufficient Database Space**: Database ran out of space during EOD
  - **Cause**: Transaction logs full or tablespace exhausted
  - **Solution**: Add disk space. Expand tablespace. Archive old transaction logs.

### Report and File Generation Errors

- **Report File Already Exists**: Cannot overwrite existing report file
  - **Cause**: Report from previous EOD still exists or is open
  - **Solution**: Delete or rename existing file. Close file if open in application.

- **Report Directory Permission Denied**: Cannot write reports to destination
  - **Cause**: User or service account lacks write permission
  - **Solution**: Grant write permissions to EOD user account on report directory

- **Invalid Report Template**: Report template is corrupted or missing
  - **Cause**: Template file deleted or modified incorrectly
  - **Solution**: Restore default template from backup or reinstall

- **Export Format Not Supported**: Selected export format not available
  - **Cause**: Report export module not installed or configured
  - **Solution**: Install required export module. Update configuration.

### Financial and Posting Errors

- **General Ledger Post Failed**: GL posting for EOD transactions failed
  - **Cause**: Invalid account, posting period issue, or GL configuration incomplete
  - **Solution**: Verify GL account exists and is active. Check posting period dates. Review GL setup.

- **Revenue Recognition Error**: Revenue recognition calculation failed
  - **Cause**: Invalid revenue codes or configuration
  - **Solution**: Verify revenue codes are valid. Check revenue recognition setup.

- **Payroll Posting Error**: Payroll information failed to post during EOD
  - **Cause**: Payroll module not configured or payroll data incomplete
  - **Solution**: Complete payroll setup. Verify employee records and rates.

### Data Integrity Errors

- **Transaction Consistency Check Failed**: Data consistency check found discrepancy
  - **Cause**: Database corruption or incomplete transaction
  - **Solution**: Run ANALYZE TABLE and rebuild indexes. If corruption suspected, restore from backup.

- **Orphaned Records Detected**: Records found without parent record
  - **Cause**: Data integrity constraint violation
  - **Solution**: Contact Oracle Support. May require data cleanup script.

- **Outstanding Items Blocking EOD**: EOD cannot complete due to outstanding transactions
  - **Cause**: Unresolved billings or adjustments
  - **Solution**: Review and post all outstanding items. Clear blocked transaction queue.

## EOD Process Steps (Normal Execution)

1. **Pre-EOD Verification**
   - Verify all guests have checked out or checkout scheduled
   - Confirm no open transactions or payment holds
   - Ensure backup destination is available and has sufficient space

2. **Run Daily Close**
   - Navigate to: Setup > Financial > Daily Close
   - Verify business date and close date
   - Click "Run Daily Close" and wait for completion
   - Verify all transactions are posted

3. **Backup Verification**
   - Confirm backup location path is correct
   - Verify sufficient disk space available (at least 50GB)
   - Check backup directory permissions

4. **Initiate EOD Process**
   - Navigate to: Setup > End of Day
   - Verify EOD parameters and settings
   - Click "Start EOD Process"
   - **Do NOT close application during EOD**

5. **Monitor Progress**
   - Watch progress indicator/log file
   - DO NOT interrupt or restart the process
   - Wait for completion message

6. **Post-EOD Verification**
   - Verify all reports completed successfully
   - Confirm backup file was created in backup directory
   - Check file size is reasonable (verify not empty/corrupted)
   - Review EOD logs for any warnings

7. **Archive Yesterday's Data**
   - If desired, archive previous day's transactions
   - Compress log files to save space

## EOD Troubleshooting Actions

### Timeout Issues
- Increase timeout in EOD configuration (Setup > System Configuration > EOD Parameters)
- Run EOD during low-traffic hours (2-5 AM is ideal)
- Optimize slow database queries: `ANALYZE TABLE; REBUILD INDEXES;`
- Reduce transaction volume by archiving older data

### Backup Failures  
- **Check disk space**: `df -h` (Linux) or `wmic logicaldisk get name,size,freespace` (Windows)
- **Add disk space**: Expand partition or mount new drive
- **Check permissions**: Verify OPERA service account has write permission
- **Verify backup path**: Confirm path exists and is accessible

### Report Failures
- **Verify path exists**: Check file path is correct and directory exists
- **Check permissions**: Ensure write permissions on report directory
- **Verify templates**: Confirm all report templates exist
- **Check printer**: If printing reports, verify default printer is available

### Posting Errors
- **GL Account Check**: Verify GL accounts exist and are active
- **Posting Period**: Check posting period is open and dates are correct
- **Outstanding Items**: Post or clear all outstanding transactions
- **Data Consistency**: Run database integrity checks

### Database Issues
- **Kill Blocking Connections**: Identify and terminate other database sessions accessing OPERA
- **Rollback Incomplete EOD**: If previous EOD was incomplete, manually rollback
- **Increase Temp Space**: Allocate more tempora/tablespace for large transactions
- **Archive Old Logs**: Remove transaction logs older than 30 days

### Common EOD Execution Sequence

```
START EOD
 Pre-check: Verify backup destination
 Pre-check: Verify GL setup
 Lock Database
 Daily Close Posting
 Post GL Transactions
 Generate Reports
   Daily Report
   Revenue Report
   Housekeeping Report
   Financial Summary
 Backup Database
 Archive Files
 Unlock Database
 END EOD (Success)
```

## EOD Parameter Tuning

- **EOD Timeout**: Default 3600 seconds (1 hour). Increase for large properties
- **Backup Chunk Size**: Larger values faster but use more memory (512 MB - 2 GB)
- **Report Output**: Specify directory for all EOD reports  
- **Archive Days**: Number of days before archiving (typically 90)
- **Posting Flag**: Enable GL posting integration if applicable

## EOD Rollback and Recovery

If EOD fails midway:
1. **Check Data State**: Query OPERA database to assess what posted
2. **Identify Failed Step**: Review EOD log to find failure point  
3. **Manual Rollback**: Reverse any partial postings through journal entries
4. **Clear EOD Lock**: May need to manually clear EOD lock flag in database
5. **Restart EOD**: Once data is consistent, restart from beginning
6. **Contact Support**: If unable to recover, call Oracle Support with EOD logs

# Export Files Troubleshooting

## Overview

Export Files module handles data export from OPERA to external systems (GDS,

## Common Export Errors - Detailed Solutions

### File System Errors

- **Error**: Error while transferring file to FTP server. Reason: Login incorrect
  - **Cause**: FTP username or password is incorrect in communication methods
  - **Solution**: Verify FTP username and password are correct. Check spelling. Test FTP credentials separately.

- **Error**: Error while transferring file. Reason: Directory path does not exist
  - **Cause**: FTP server folder path is incorrect or doesn't exist
  - **Solution**: Verify FTP directory path syntax and that directory exists on server. Contact FTP administrator.

- **Error**: Export Destination Not Found
  - **Cause**: Export file path does not exist or is inaccessible
  - **Solution**: Verify path syntax (use full path like C:\Export\data.csv). Create destination directory if missing.

- **Error**: Export File Permission Denied
  - **Cause**: User or system account lacks write permission on destination folder
  - **Solution**: Grant write permissions to OPERA service account on export directory. Check NTFS permissions.

- **Error**: File Already Exists - Overwrite Denied
  - **Cause**: Export file already exists and overwrite flag is not set
  - **Solution**: Delete existing file or enable overwrite option in export configuration

- **Error**: Export Timeout
  - **Cause**: Export took too long and was cancelled (usually > 3600 seconds)
  - **Solution**: Reduce data volume being exported. Run during off-peak hours. Increase timeout if possible.

### Data Format Errors

- **Error**: Export Format Invalid
  - **Cause**: Selected export format not supported
  - **Solution**: Choose valid export format from dropdown (CSV, XML, Excel, Tab-delimited, etc.)

- **Error**: Export Data Incomplete
  - **Cause**: Not all required data fields available for export
  - **Solution**: Verify all records have required information populated. Check field mapping.

- **Error**: No Data Found to Read for Group [Number]
  - **Cause**: Export file sent but EXP_DATA_TABLE has empty data record
  - **Solution**: Verify source records contain data. Check table space allocation. Contact PMS team to research.

- **Error**: Invalid Field Format
  - **Cause**: Field contains data in wrong format for export specification
  - **Solution**: Correct source data format before export. V example: ensure dates are in YYYY-MM-DD format.

- **Error**: Character Encoding Mismatch
  - **Cause**: Text contains characters not supported by export encoding
  - **Solution**: Remove special characters or select UTF-8 encoding if available.

### Configuration Errors

- **Error**: Export Configuration Missing
  - **Cause**: Export batch not configured or setup incomplete
  - **Solution**: Setup export in Setup > Channels > Export Files > Configuration

- **Error**: Export File Format Template Missing
  - **Cause**: Custom export template deleted or not found
  - **Solution**: Restore template from backup or create new template

- **Error**: Invalid Field Mapping
  - **Cause**: Source field mapped to invalid or non-existent destination field
  - **Solution**: Review field mappings. Correct or remove invalid mappings.

- **Error**: Export Recipient Not Defined
  - **Cause**: Export destination system/contact not configured
  - **Solution**: Define export recipient in Setup > Export Files > Recipients

### External System Integration Errors

- **Error**: FTP Connection Failed
  - **Cause**: Cannot connect to FTP server (network issue, server down, firewall blocking)
  - **Solution**: Verify FTP server is online. Check network connectivity. Review firewall rules. Test with `ftp` command.

- **Error**: FTP Login Failed
  - **Cause**: Invalid FTP credentials or account locked
  - **Solution**: Verify username/password. Check if FTP account is active and not locked.

- **Error**: Pegasus Connection Error
  - **Cause**: Connection to Pegasus GDS failed
  - **Solution**: Verify Pegasus is online. Check Pegasus credentials. Verify OXI configuration.

- **Error**: OTA API Connection Failed
  - **Cause**: Cannot connect to OTA provider's API
  - **Solution**: Verify OTA endpoint URL. Check OTA credentials. Test network connectivity to OTA.

- **Error**: Export Recipient System Offline
  - **Cause**: Destination system not available to receive export
  - **Solution**: Verify destination system is online. Retry export. Contact destination system administrator.

## Export File Status and Management

### Export File Statuses
- **PENDING**: File queued for export, awaiting processing
- **IN_PROGRESS**: File currently being exported
- **SUCCESSFUL**: File successfully exported to destination
- **ERROR**: Export failed - see error details
- **RETRY**: Automatic or manual retry in progress
- **ARCHIVED**: File exported and archived (typically after 30-90 days)

### Export File Error Handling Workflow

1. **Locate Failed Export**: Setup > Channels > Export Files > Error Handling
2. **Review Error Details**: Click error to view message, reason, and solution
3. **Identify Root Cause**: Determine if configuration, data, or connectivity issue
4. **Correct Issue**:
   - For configuration errors: Update setup
   - For data errors: Correct source data
   - For connectivity: Verify destination system/network
5. **Reprocess Export**: Locate failed export by ID and click "Retry" or "Reprocess"
6. **Monitor Status**: Check export status changes to PENDING  SUCCESSFUL
7. **Verify Destination**: Confirm file received and processed by destination system

## Export File Configuration

### Required Configuration for Export

1. **Setup > Channels > Export Files > Configuration**
   - Define export file format (CSV, XML, etc.)
   - Map OPERA fields to export fields
   - Define export frequency (immediate, scheduled, manual)

2. **Destination Configuration**
   - Specify file destination (FTP, local folder, email, API)
   - For FTP: Save path, server, username, password/key
   - For file system: Full path with write permissions
   - For API: Endpoint URL, credentials, authentication method

3. **Data Selection**
   - Select data source (reservations, rates, inventory, etc.)
   - Define filter criteria (date range, property, status, etc.)
   - Map required and optional fields

4. **File Naming Convention**
   - Auto-generate name pattern (e.g., HOTEL_YYYYMMDD.csv)
   - Or specify manual name

5. **Retry Configuration**
   - Set retry count (typically 3-5 attempts)
   - Set retry interval (delay between retries)
   - Set max wait time before giving up

## Export Troubleshooting Checklist

1. **Destination System Checks**
   - Verify destination system is online and accepting connections
   - Test connectivity: `ping hostname` or `telnet host port`
   - Confirm destination has capacity (storage space, quota)

2. **Credentials and Authentication**
   - Verify FTP/API credentials are correct
   - Confirm credentials haven't expired or been reset
   - Check if account is locked or disabled

3. **File and Path Checks**
   - Verify destination path exists and is accessible
   - Check for trailing spaces or hidden characters in path
   - Ensure write permissions granted to OPERA service account
   - Confirm sufficient disk space on destination (at least 10% free)

4. **Data Quality Checks**
   - Verify all required source fields have data
   - Check data format matches export specification
   - Ensure no special characters or encoding issues
   - Review for NULL values in mandatory fields

5. **Configuration Checks**
   - Verify export configuration includes all required mappings
   - Confirm export recipients are defined
   - Check file naming doesn't contain invalid characters
   - Ensure export format matches destination requirements

6. **Network and Firewall**
   - Verify firewall allows outbound on export port (FTP=21, SFTP=22, HTTPS=443)
   - Check if proxy authentication required for API exports
   - For OTA/API, verify SSL certificates are valid

7. **Log File Review**
   - Check export logs: Setup > Channels > Export Files > View Logs
   - Look for timeout messages, connection refused, or permission denied
   - Search for specific error codes matching error list above

8. **Test Export**
   - Create test export with small dataset
   - Verify file created successfully
   - Confirm file received at destination
   - Validate file contents for accuracy

## Manual Export in Case of Failures

If automated export fails repeatedly:
1. Navigate to source data module (Rates, Availability, etc.)  
2. Select records to export
3. Right-click > Export Data
4. Choose format and destination
5. Save file
6. Manually transfer to destination (SCP, FTP client, email, etc.)
7. Notify IT/GDS team of manual export
8. Update Status in OPERA to mark as processed

# POS Terminal Errors & Products

Understanding Common POS Terminal Error Codes: Troubleshooting for Smooth Transactions

In today's digital world, Point of Sale (POS) systems have become an integral part of business operations. These systems enable businesses to process transactions efficiently and provide a seamless checkout experience for customers. However, like any technology, POS terminals can encounter errors that disrupt payment processing and potentially impact customer satisfaction. In this section we explore some common POS terminal error codes and provide troubleshooting tips.

## Error Code 100 - Connection Failure
- Check network connections: Ensure all cables and connections are secure, and the router or modem is functioning correctly.
- Restart the terminal: Sometimes a simple restart can fix temporary connectivity issues.
- Verify IP settings: Ensure the terminal's IP address is correct and matches the network's settings.

## Error Code 200 - Card Declined
- Ask the customer for an alternate payment method: Prompt the customer to use a different credit or debit card.
- Verify card details: Double-check that the card information entered into the POS terminal is correct, including the card number, expiration date, and CVV code.
- Contact the payment processor: In some cases the decline may be due to an issue on the payment processor's end.

## Error Code 300 - Printer Error
- Check paper and ink levels: Ensure there is sufficient paper and ink in the printer to complete the print job.
- Clear any paper jams: If the printer is jammed, carefully remove the stuck paper to restore normal printing functionality.
- Restart the printer: A simple restart can sometimes resolve temporary printer errors.

## Error Code 400 - Transaction Timeout
- Check network connection: Ensure the terminal is connected to a stable and reliable network.
- Retry the transaction: Attempt the transaction again, as the previous timeout might have been due to a temporary issue.
- Contact support: If the timeouts persist, contact the POS terminal provider's support team for further assistance.

## Error Code 500 - System Error
- Restart the terminal: A simple restart can sometimes resolve temporary system glitches.
- Update software and firmware: Check for any available updates for the POS terminal's software and firmware.
- Contact technical support: If the system error persists, reach out to the POS terminal manufacturer's technical support for guidance.

## Error Code 600 - Invalid Card
- Inspect the card: Check for any visible damage to the card's magnetic strip or chip.
- Clean the card reader: Gently clean the card reader with a soft, lint-free cloth to remove any dirt or debris.
